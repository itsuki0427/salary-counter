# Requirements: 秒給与動的計算モデル

## 概要

秒給を**毎秒動的に計算**し、ユーザーが給与計算ルールを**組み立て可能**にする仕様。

現在のモデル（秒給固定）では「給与日時点での理論値」に過ぎず、実作業量に応じた給与増減をリアルタイムに反映できない。新モデルは、秒給を「実際の給与増減を秒単位で逆算した数字」として扱い、毎秒の行動が実給与にいくら貢献しているかを正確に示す。

## 優先度

1. **精度**: 秒単位での給与計算が正確であること（絶対優先）
2. **拡張性**: 残業ルール（時間帯・曜日・倍率）が後で追加できること
3. **直感性**: UI で「今いくら稼いでいるのか」が一目でわかること
4. **複雑度**: ロジックがシンプル＆保守しやすいこと（優先度最低）

## Functional Requirements (FR)

### FR-1: 秒給動的計算

**目的**: 秒給をリアルタイムで再計算し、給与日までの残り勤務秒数の変化に追従する。

```
秒給(t) = (給与日に受け取る給与 - すでに計上された給与) ÷ (実勤務秒 + 予定残り秒)
```

- **給与日に受け取る給与**: 月給 + 発生した全残業代（見なし超過、時間外含む）
- **実勤務秒**: 月初からの実作業秒数
- **予定残り秒**: 給与日までの予定残り勤務秒数
- **秒給**: 小数点第4位まで正確に計算

#### AC-1.1: 見なし残業内での秒給低下

- **Given**: 月給 300,000円、見なし時間 45h、月勤務時間 160h
- **When**: 月中盤（実作業 30h）にて秒給を表示
- **Then**: 秒給 = 300,000 ÷ ((30×3600) + (160-30)×3600) = 0.52... 円/秒

#### AC-1.2: 見なし超過時の秒給上昇

- **Given**: 同上。見なし超過分の残業代が秒給計算に反映される
- **When**: 月中盤（実作業 50h、うち超過 5h）にて秒給を表示
- **Then**: 秒給 = (300,000 + 超過代) ÷ ((50×3600) + (160-50)×3600) > AC-1.1 の秒給

#### AC-1.3: 時刻変更による秒給自動更新

- **Given**: 22-5時に自動的に 1.25倍が適用される
- **When**: 21:59:59 から 22:00:00 に時刻が進む
- **Then**: 秒給が 1.25倍に自動更新される（ユーザー操作不要）

### FR-2: ルール組み立て可能な給与計算

**目的**: ユーザーが「見なし超過」「深夜労働」「休日労働」などのルールをカスタマイズでき、複数ルール複合時の倍率を定義できる。

#### ルール構造

```typescript
type Rule = {
  id: string;
  name: string;              // 「見なし超過」「深夜」など
  type: 'assumed-overtime' | 'time-range' | 'weekday' | 'custom';
  condition: {
    assumedHours?: number;   // 見なし超過: 45h
    startHour?: number;      // 時間帯: 22
    endHour?: number;        // 時間帯: 5
    dayOfWeek?: number;      // 曜日: 0=日
  };
  multiplier: number;        // 1.25 など
  priority: number;          // 優先度（複合時）
  enabled: boolean;          // 有効/無効
};
```

#### AC-2.1: デフォルトルール設定

- **Given**: アプリを初期化したユーザー
- **When**: UI でルール設定画面を開く
- **Then**: 以下の3つのルールがデフォルト有効で表示される：
  - 見なし超過（見なし時間超過時、倍率 1.0）
  - 深夜労働（22-5時、倍率 1.25）
  - 休日労働（日曜、倍率 1.25）

#### AC-2.2: カスタムルール追加

- **Given**: ルール設定画面でユーザーが「+ ルール追加」を操作
- **When**: 新ルール（例：「金曜 18-22時は 1.1倍」）を定義し、「保存」をタップ
- **Then**: ルール一覧に新ルールが追加され、以降の秒給計算に反映される

#### AC-2.3: ルール有効/無効切り替え

- **Given**: ルール一覧に「深夜労働」ルールが表示されている
- **When**: ルール横のトグルを OFF にする
- **Then**: 深夜時間帯でも秒給が 1.25倍にならず、通常計算される

### FR-3: 複数ルール複合時の優先度制御

**目的**: 「日曜の深夜労働」など、複数ルールが同時にマッチする場合、ユーザーが優先度を指定でき、それに従って倍率が計算される。

#### AC-3.1: 複数ルール同時マッチ時の倍率決定

- **Given**: 日曜の 23時（「日曜」1.25倍 + 「深夜」1.25倍の両ルールマッチ）
- **When**: ルール設定で両ルールの優先度が設定されている（例：日曜=優先度1、深夜=優先度2）
- **Then**: 優先度 1 の「日曜」1.25倍が適用される（最大値優先）

#### AC-3.2: 優先度変更による秒給再計算

- **Given**: AC-3.1 の状態で、優先度を「深夜=優先度1」に変更
- **When**: ルール設定を保存
- **Then**: 日曜 23時の秒給が「深夜」1.25倍で再計算される（優先度反映）

### FR-4: 入力インターフェース拡張

**目的**: 秒給動的計算に必要な入力項目（給与日、見なし時間、ルール設定）を UI に追加する。

#### AC-4.1: 給与日入力フィールド

- **Given**: UI に「給与日」という新しい入力フィールドが追加されている
- **When**: ユーザーが日付を選択（例：「15日」）
- **Then**: その日付以降の「残りの勤務秒数」計算に使用される

#### AC-4.2: 見なし時間入力フィールド

- **Given**: 既存の「見なし時間」フィールドが提供されている
- **When**: ユーザーが「45」と入力
- **Then**: 秒給計算で見なし時間 45h が基準として使用される

#### AC-4.3: ルール管理画面

- **Given**: UI に「ルール設定」という新しい画面セクションがある
- **When**: ユーザーが「ルール設定」をタップ
- **Then**: ルール一覧表示、ルール追加、ルール編集、優先度変更ができる画面が開く

## Non-Functional Requirements (NFR)

### N-1: 精度

- 秒給は小数点第4位まで計算し、丸め誤差を最小化する
- 給与日時点での「実勤務秒数」と「秒給の累積」が月給と一致する（誤差 ±1円以内）

### N-2: パフォーマンス

- 秒給の再計算は **100ms 以内**（毎秒の更新ループで遅延なし）
- ルール判定は **10ms 以内**（時刻変更時の判定）

### N-3: 保守性

- ルール定義とルール適用ロジックを分離する（ルール追加時は定義だけで充分）
- コードの循環的複雑度 <= 10（関数単位）

### N-4: 拡張性

- 新規ルール型（`type: 'custom'`）を追加する場合、既存コードの修正を最小化する
- ルール優先度アルゴリズムの変更（「最大値優先」→「加算」等）を容易にするため、優先度制御を独立した関数に分離する

### N-5: エラーハンドリング

- 給与日が「今日」より過去の場合、警告を表示し、計算を続行しない
- ルール条件に矛盾がある場合（例：startHour=22, endHour=5 の時間帯が逆）、バリデーション エラーを表示

## Scope

### In Scope
- 秒給動的計算ロジックの実装
- ルール組み立て UI の実装
- テスト（Unit / Integ）
- 既存の「開始/停止/リセット」機能との統合

### Out of Scope
- ユーザーアカウント管理（ルール設定のクラウド保存）
- 給与日自動更新（カレンダー API 連携）
- 複数ユーザー対応
- 多言語化

## 用語集

| 用語 | 定義 |
|------|------|
| 秒給 | 1秒あたりの給与額（円/秒） |
| 見なし時間 | 月給に含まれる想定勤務時間（例：45h） |
| 見なし超過 | 実勤務時間が見なし時間を超えた部分 |
| ルール | 給与倍率を適用する条件セット（例：「日曜は 1.25倍」） |
| 優先度 | 複数ルール複合時に、どのルールを優先するかの指標（数値） |
| 給与日 | 月給が振込される日付（例：月末日） |
| 実勤務秒 | 月初からの累積実作業時間（秒単位） |
| 予定残り秒 | 給与日までの予定勤務秒数 |
